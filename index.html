Redux (The Old Way) 😓

// 45+ lines of boilerplate...
const INCREMENT = 'INCREMENT';
const incrementAction = () => ({ 
  type: INCREMENT 
});
 
const counterReducer = (state = 0, action) => {
  switch(action.type) {
    case INCREMENT: 
      return state + 1;
    default: 
      return state;
  }
};
 
const store = createStore(counterReducer);
 
// In component
const dispatch = useDispatch();
const count = useSelector(state => state);
 
<button onClick={() => dispatch(increment())}>
  {count}
</button>
 
// Testing? Don't ask... 😭

Pumped Functions (The New Way) ✨

// Just 8 lines. That's it!
const counter = provide(() => 0);
const increment = derive(counter.static, 
  ctl => () => ctl.update(n => n + 1)
);
 
// In component
const [count, inc] = useResolves(
  counter.reactive, 
  increment
);
 
<button onClick={inc}>
  {count}
</button>
 
// Testing? One line!
createScope(preset(counter, 5));

That’s 80% Less Code. Seriously. 🤯

Why Pumped Functions?

🎯 Problem: State Management Sucks

Every state library makes you choose between:

  • Simple but Limited (Context API)
  • Powerful but Complex (Redux)
  • Nice but Hard to Test (Zustand)

✨ Solution: What If It Was Just Functions?

// Your entire app state
const user = provide(() => ({ name: 'John' }));
const theme = provide(() => 'light');
 
// Update it
const setTheme = derive(theme.static, ctl => 
  (newTheme) => ctl.update(newTheme)
);
 
// React to changes
const greeting = derive(
  [user.reactive, theme.reactive],
  ([user, theme]) => `Hello ${user.name}! Theme: ${theme}`
);
 
// Test it
const scope = createScope(
  preset(user, { name: 'Test' }),
  preset(theme, 'dark')
);

That’s it. No providers. No reducers. No actions. Just functions.
